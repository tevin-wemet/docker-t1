# 사용자 시나리오

> 최종 수정: 2026-04-20

## 페르소나

- **일반 사용자 (U)**: 가족 구성원, 자신의 일기를 비공개로 기록·관리
- **관리자 (A)**: NAS 운영자, 사용자 계정과 시스템을 관리

---

## 시나리오 U-1. 신규 회원가입 및 첫 일기 작성

1. 사용자가 브라우저로 `http://nas.local:3000` 접속
2. 로그인 페이지에서 "회원가입" 클릭
3. 아이디 / 비밀번호 입력 후 제출
4. 가입 성공 시 자동 로그인, 일기 목록(빈 화면)으로 이동
5. "새 일기 작성" 클릭 → 제목/날짜/본문 입력 후 저장
6. 목록에 방금 작성한 일기 1건이 표시됨

## 시나리오 U-2. 일기 수정 / 삭제

1. 로그인 상태에서 일기 목록 진입
2. 특정 일기 클릭 → 상세 보기
3. "수정" 버튼 → 본문 수정 후 저장 → 상세 화면에 반영
4. "삭제" 버튼 → 확인 다이얼로그 → 승인 시 목록으로 돌아오고 해당 항목 사라짐

## 시나리오 U-3. 로그아웃 및 재로그인

1. 상단 "로그아웃" 클릭 → 세션 파기, 로그인 페이지로 이동
2. 다른 기기(휴대폰)에서 동일 계정 로그인
3. 이전에 작성한 일기가 동일하게 표시됨 (영속성 확인)

## 시나리오 U-4. 타인의 일기 접근 차단

1. 사용자 B로 로그인한 상태에서 사용자 A의 일기 ID 를 URL 로 직접 입력
2. 403 또는 404 로 차단되어야 함

---

## 시나리오 A-1. 관리자 로그인

1. `admin` 계정으로 로그인 → 사용자 화면 대신 관리자 대시보드로 이동 (또는 관리자 메뉴 노출)

## 시나리오 A-2. 사용자 관리

1. 관리자 대시보드 → "사용자" 탭
2. 사용자 목록에서 특정 사용자 선택
3. 비밀번호 초기화 또는 계정 삭제 수행
4. 계정 삭제 시 해당 사용자의 일기도 함께 삭제됨 (CASCADE)

## 시나리오 A-3. 전체 일기 조회 및 삭제

1. 관리자 대시보드 → "일기" 탭
2. 전체 사용자 일기 목록(작성자명 표시) 확인
3. 부적절한 콘텐츠 삭제 가능

---

## 네트워크 검증 시나리오 N-1

1. NAS 내부망 PC 에서 접속 → 정상 동작 확인
2. 외부망(휴대폰 LTE) 에서 DDNS 주소로 접속 → 정상 동작 확인
3. 컨테이너 재시작 후 데이터 유지 확인
