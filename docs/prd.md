# PRD — 시놀로지 NAS 일기장 시스템

> 최종 수정: 2026-04-20

## 1. 배경 및 목적

시놀로지 NAS 의 Docker 환경을 활용해 가정 내·외부 네트워크에서 접근 가능한 개인 일기장 서비스를 구축한다. 본 프로젝트의 주된 목적은 **Docker 컨테이너가 NAS 네트워크 환경에서 정상 동작하는지 검증**하는 것이며, 일기장 기능은 검증을 위한 최소 기능 샘플이다.

## 2. 범위 (MVP)

### 포함
- 사용자 회원가입 / 로그인 / 로그아웃
- 일기 작성 · 조회(목록/상세) · 수정 · 삭제
- 사용자 화면 (일반 회원)
- 관리자 화면 (사용자 관리, 전체 일기 조회/삭제)
- 단일 Docker 컨테이너 배포 (`docker-compose.yml` 제공)

### 제외
- 이메일 인증, 비밀번호 재설정
- 이미지/파일 첨부
- 태그/검색/공유 기능
- 멀티 테넌시, SSO
- 모바일 전용 앱

## 3. 기술 스택 (제안)

| 영역 | 선택 | 이유 |
|------|------|------|
| 런타임 | Node.js 20 LTS | NAS 자원 대비 가벼움 |
| 웹 프레임워크 | Express | 학습 비용 낮음 |
| 템플릿 | EJS (서버 렌더링) | SPA 불필요, 단순함 |
| DB | SQLite | 별도 컨테이너 없이 파일 1개로 운영 |
| 인증 | express-session + bcrypt | 세션 쿠키 기반 |
| 컨테이너화 | Docker + docker-compose | 시놀로지 표준 |

> 대안: Python + FastAPI + SQLite, 또는 PHP + MariaDB. 최종 스택은 사용자 승인 후 확정.

## 4. 비기능 요구사항

- **접근성**: NAS 내부망(LAN), VPN 연결, 시놀로지 QuickConnect 세 가지 경로 모두에서 접근 가능해야 함
- **지속성**: SQLite DB 파일은 호스트 볼륨에 마운트하여 컨테이너 재시작 시 유지
- **보안**: 비밀번호는 bcrypt 해시, 세션 쿠키 `HttpOnly` + `SameSite=Lax`
- **성능**: 동시 사용자 10명 이하 기준 (개인/가족용)
- **관측성**: 기본 `stdout` 로그만 제공 (MVP)

## 5. 주요 화면

### 5.1 사용자 화면
1. 로그인 / 회원가입
2. 일기 목록 (내 일기만, 날짜 역순)
3. 일기 작성/수정 폼
4. 일기 상세 보기

### 5.2 관리자 화면
1. 관리자 로그인 (동일 로그인 페이지, `role=admin` 시 진입)
2. 사용자 목록 (비밀번호 초기화, 삭제)
3. 전체 일기 목록 (작성자 필터, 삭제)

### 5.3 관리자 초기 계정 생성

서버 기동과 별개로 CLI 커맨드로 수동 생성한다.

```bash
npm run seed-admin -- --username admin --password <비밀번호>
```

## 6. 배포 개요

```
시놀로지 NAS
 └─ Docker
     └─ diary-app (Node.js 컨테이너)
         ├─ 볼륨: /data → SQLite DB 파일
         └─ 포트: 3000 (호스트 임의 포트로 매핑)
```

## 7. 성공 기준

- NAS 외부(다른 PC/모바일)에서 로그인/일기 CRUD 동작 확인
- 컨테이너 재시작 후에도 데이터 유지
- 관리자/사용자 권한 분리가 UI 레벨에서 확인됨
